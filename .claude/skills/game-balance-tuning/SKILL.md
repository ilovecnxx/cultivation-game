# Game Balance Tuning — 九转修仙项目

分析九转修仙游戏的平衡性数值，检测异常、通货膨胀和曲线问题，给出具体调校建议。

## 数据来源

游戏数值定义于以下文件：
- **`docs/BALANCE.md`** — 核心数值平衡表（怪物属性、技能、丹药、区域）→ 权威来源
- **`database/mysql/`** — 数据库表结构（realm_config, technique_templates 等）
- **`services/combat/internal/data/`** — 战斗相关静态数据 JSON
- **`services/cultivation/internal/data/`** — 修炼相关静态数据 JSON
- **`shared/models/`** — 共享模型（realm.go, combat.go, item.go, player.go）

始终以 `docs/BALANCE.md` 为主进行数值分析，代码中的数据文件作为补充验证。

---

## 分析维度

### 1. 境界修炼曲线分析

检查 `realm_config` 中定义的境界体系：

**关键指标**：
- 境界 HP 倍率：相邻境界 HP 比值是否平滑递增（建议 1.5× ~ 3.0×）
- 攻击力倍率：相邻境界攻击力比值
- 经验值需求：下一级所需经验 / 当前级总经验（建议 1.2× ~ 2.0×）
- 突破概率：随着境界提升，基础突破成功率应逐步降低

**问题检测**：
- 某个境界到下一个境界的属性跃升过高（>5×）→ 可能导致该境界卡住大量玩家
- 属性增长太慢（<1.2×）→ 玩家感觉不到变强
- 经验曲线在某处出现"断崖"→ 升级时间从 1 小时突然变成 10 小时

**输出格式**：

```
## 境界修炼曲线报告

| 境界 | HP范围 | 攻击范围 | HP倍率 | 攻击倍率 | 经验需求 | 评估 |
|------|--------|---------|--------|---------|---------|------|
| 凡人 | 100 | 10 | - | - | 100 | 基准 |
| 练气 | 500-2000 | 30-80 | 5.0× | 3.0× | 500 | ⚠️ HP倍率偏高 |
| 筑基 | 2000-5000 | 80-150 | 2.5× | 2.0× | 2000 | ✅ |
| ... | ... | ... | ... | ... | ... | ... |

**发现的问题**：
1. 练气→筑基的HP倍率(5.0×)远超推荐范围，建议调整为2.5×
2. ...
```

### 2. 战力系统分析

检查战力计算公式中各属性的权重和贡献：

**关键检查**：
- HP、攻击、防御、速度的权重是否合理
- 是否有某个属性对战力的贡献过高，导致不平衡（如速度权重高导致"唯快不破"）
- 五行相克系数（克制 +30%，被克制 -20%）是否在生产环境中验证过

**分析步骤**：
1. 收集各境界典型玩家的四维属性
2. 应用战力公式计算战力值
3. 绘制战力-境界曲线，检查是否平滑
4. 分析各属性对最终战力的贡献占比

### 3. 经济系统分析

**关键指标**：
- 灵石获取率（每小时各境界能获取多少灵石）
- 灵石消耗率（修炼/装备强化/丹药的灵石消耗）
- 物价指数（关键物品（丹药、装备）的灵石价格）
- 通货膨胀风险（灵石总产出 > 总消耗？货币是否会无限膨胀？）

**检查清单**：
- 怪物灵石掉落 ÷ 怪物刷新时间 = 境界灵石获取率
- 丹药价格 vs 丹药成本（材料 + 炼制时间）
- 装备价格 vs 强化费用
- 交易行税率是否足够回收流通货币（建议 5%~10%）

**输出格式**：

```
## 经济系统报告

### 灵石产出（每小时）
| 境界 | 刷怪产出 | 任务产出 | 交易所得 | 合计 |
|------|---------|---------|---------|------|
| 练气 | 200 | 100 | 50 | 350 |
| ... | ... | ... | ... | ... |

### 灵石消耗（每小时）
| 境界 | 修炼消耗 | 丹药消耗 | 装备强化 | 合计 |
|------|---------|---------|---------|------|

### 净流入/流出
| 境界 | 产出 | 消耗 | 净额 | 通胀风险 |
|------|------|------|------|---------|

**发现的问题**：
1. 元婴期灵石净流入过高(+500/h)，长期会导致通货膨胀
2. ...
```

### 4. 怪物属性递进分析

**关键检查**：
- HP 递进比例：相邻境界怪物 HP 中位数比率（建议 2.0× ~ 4.0×）
- 攻击递进比例：应略低于 HP 增长（否则战斗变成互秒）
- 防御递进比例：应与攻击增长相匹配，维持 TTK（Time-to-Kill）稳定
- Boss 强度：Boss 属性应约为同级普通怪物的 3×~8×
- 经验/灵石掉落效率：每点 HP 能获取的经验值是否稳定

**输出格式**：

```
## 怪物属性递进报告

| 境界 | 怪物HP中位数 | 怪物攻击中位数 | HP倍率 | 攻击倍率 | 经验/HP | 评估 |
|------|-------------|---------------|--------|---------|---------|------|
| 练气 | 135 | 17 | - | - | 0.14 | 基准 |
| 筑基 | 600 | 50 | 4.4× | 2.9× | 0.08 | ⚠️ 经验/HP下降严重 |
| 金丹 | 3,500 | 120 | 5.8× | 2.4× | 0.03 | ⚠️ HP倍率过高 |
| ... | ... | ... | ... | ... | ... | ... |

**发现的问题**：
1. 筑基→金丹的HP倍率(5.8×)过高，建议降至3.0×-4.0×
2. 经验/HP比率从练气(0.14)降到金丹(0.03)，掉落严重。金丹期刷怪效率太低。
```

### 5. 技能伤害平衡

**关键检查**：
- 同境界技能伤害系数是否在合理范围（建议 80% ~ 200% 普通攻击）
- 高境界技能是否严格优于低境界技能
- CD 与伤害是否反比关系（CD 越长伤害越高）
- 五行相克加成是否造成特定属性过强

### 6. 丹药经济平衡

**关键检查**：
- 丹药效果 vs 炼制成本（灵石效率）
- 永久属性丹药与临时丹药的性价比
- 高境界丹药是否比低境界丹药有足够的提升

## 工作流程

1. **阅读 `docs/BALANCE.md`** — 获取最新数值
2. **确定分析范围** — 用户可能只想检查某个系统（如"检查怪物平衡"），或做全面检查
3. **收集数据** — 从 BALANCE.md 和相关代码 JSON 文件中提取数值
4. **计算指标** — 应用上述分析维度计算关键指标
5. **生成报告** — 使用指定格式输出，标注 ✅/⚠️/❌
6. **给出建议** — 每个问题附具体数值调整建议（调整到什么值，为什么）

## 设计原则提醒

以下原则来自 `docs/BALANCE.md` 和 `docs/DESIGN.md`，调校时应遵循：

1. **可感知的成长**：每次升级/突破应该有可感知的战力提升（至少 20%）
2. **平滑的难度曲线**：玩家应该在当前境界刷当前境界的怪物感到适中（TTK 30s~120s）
3. **正向经济循环**：灵石产出略高于基础消耗（净流入 10%~20%），让玩家能积累财富
4. **技能多样性**：没有"必须带"的技能，不同流派（暴击流、dot流、控制流）都有可玩性
5. **五行平衡**：五种属性没有明显强弱，克制关系起到战术作用而非决定性
6. **社交激励**：组队/双修/宗门玩法的效率要高于单刷（高 20%~50%）

## 数值基准参考

| 指标 | 推荐范围 | 说明 |
|------|---------|------|
| 境界HP倍率 | 1.5× ~ 3.0× | 相邻境界HP比值 |
| 境界攻击倍率 | 1.3× ~ 2.5× | 应低于HP倍率 |
| TTK (击杀同级怪) | 30s ~ 120s | 时间过长则无聊 |
| Boss vs 普通怪 | 3× ~ 8× | Boss属性倍数 |
| 丹药效率 | 0.5 ~ 2.0 | 丹药增加的战力 / 消耗的灵石 |
| 灵石净流入率 | +10% ~ +30% | 产出 - 基础消耗 |
| 强化成功率 | 100% → 30% | 随着强化等级从100%逐渐降低 |
| 突破成功率 | 90% → 10% | 随着境界从90%逐渐降低 |
